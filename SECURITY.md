# Security Policy

## Reporting a Vulnerability

**Please do not open a public issue for security vulnerabilities.**

Instead, report them privately:

**Email:** manuel.turpin@gmail.com

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial Assessment:** Within 1 week
- **Resolution:** Depends on severity

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | ✅ Yes    |
| < 1.0   | ❌ No     |

## Security Best Practices

When using ACT Framework:

- Keep your Claude Code installation updated
- Don't commit `.epct/` folder if it contains sensitive project data
- Review generated code before deploying to production

## Recognition

We appreciate responsible disclosure. Contributors who report valid security issues will be acknowledged in release notes (unless they prefer to remain anonymous).
