## How to Use This File

This file captures **runtime decisions** made during task execution — quick choices that don't warrant a full ADR but should be recorded for context.

### When to Log Here
- Choosing between two implementation approaches
- Selecting a library or tool for a specific task
- Deciding to skip or defer something
- Accepting a trade-off
- Resolving an ambiguity in the spec

### When NOT to Log Here
- Architectural decisions → Use ADR workflow (`plugin/workflows/adr/`)
- Bugs found → Use ISSUES.md (Deviation Rule 5)
- Task progress → Use `.act/progress.md`

## Decision Format

```markdown
### DEC-NNN: Brief Title

- **Date:** YYYY-MM-DD
- **Context:** Why this decision was needed (1-2 sentences)
- **Decision:** What was decided
- **Alternatives considered:** What else was considered (brief)
- **Rationale:** Why this option was chosen
- **Phase:** Current ACT phase when decided
```
