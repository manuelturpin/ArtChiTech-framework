# SPEC: GitHub Integration

> Synchronisation bidirectionnelle entre ACT et GitHub Issues/PR

## Objectif

Permettre la gestion de projet intégrée entre l'état local ACT (`.act/plan.md`, `progress.md`) et GitHub (Issues, Pull Requests, Labels).

**Bénéfices :**
- Visibilité du projet sur GitHub pour l'équipe
- Traçabilité des phases via Issues
- Historique des décisions dans les PR
- Collaboration facilitée

---

│   ├── plan.md    │ ◄─────► │   ├── Issues     │

## Mapping plan.md → Issues

### Structure de plan.md

```markdown
## Phases

### Phase 1: Discovery ✅
- [x] Define problem statement
- [x] Identify stakeholders

### Phase 2: Strategy 🔄
- [x] Create roadmap
- [ ] Define milestones

### Phase 3: Design ⏳
- [ ] Architecture diagrams
- [ ] API specification
```

### Extraction

| Élément plan.md | GitHub Issue |
|-----------------|--------------|
| `### Phase N: Name` | Issue title: `[Phase N] Name` |
| `✅` (complete) | Issue closed |
| `🔄` (in-progress) | Issue open + label `in-progress` |
| `⏳` (pending) | Issue open + label `pending` |
| `- [x] task` | Checkbox dans le body (checked) |
| `- [ ] task` | Checkbox dans le body (unchecked) |

## Mapping progress.md → Issue Comments

Chaque entrée de session dans `progress.md` peut être synchronisée comme commentaire sur l'Issue correspondante.

```markdown
## Session 2026-02-02 14:30

### Completed
- Implemented login endpoint
- Added unit tests (coverage: 87%)

### Notes
- JWT preferred over sessions
```

↓ Devient commentaire sur Issue `[Phase 4] Development` :

```markdown
## Session Update: 2026-02-02 14:30

**Completed:**
- ✅ Implemented login endpoint
- ✅ Added unit tests (coverage: 87%)

**Notes:**
- JWT preferred over sessions

---
*Synced by ACT Framework*
```

## Format des Pull Requests

### Template PR

```markdown
## 🎯 Objective

[Phase N] - Description of what this PR achieves

## 📋 Changes

- Change 1
- Change 2

## 🔗 Related Issues

Closes #XX (Phase N Issue)

## ✅ Checklist

- [ ] Tests pass
- [ ] Documentation updated
- [ ] Code reviewed

## 📊 ACT Context

**Phase:** N/7
**Progress:** XX%
**Session:** YYYY-MM-DD HH:MM

---
*Generated by ACT Framework*
```

### Liaison automatique

Les commits avec le format `feat(act): [Phase N.X] Message` sont automatiquement liés à l'Issue correspondante.

## Commandes

### /act:sync-github

```bash
/act:sync-github [--push | --pull] [--dry-run] [--labels]
```

| Option | Description |
|--------|-------------|
| `--push` | Pousse l'état local vers GitHub Issues |
| `--pull` | Récupère l'état des Issues vers local |
| `--dry-run` | Affiche les changements sans les appliquer |
| `--labels` | Crée/met à jour les labels automatiquement |
| (aucune) | Mode interactif, affiche le diff |

### Exemples

```bash
# Voir les différences
/act:sync-github

# Pousser l'état local
/act:sync-github --push

# Pousser avec création des labels
/act:sync-github --push --labels

# Simuler sans appliquer
/act:sync-github --push --dry-run

# Récupérer l'état GitHub
/act:sync-github --pull
```

## Configuration

Dans `.act/config.yaml` :

```yaml
github:
  enabled: true
  repo: owner/repo  # Auto-detected from git remote if not set
  sync:
    issues: true        # Sync phases as issues
    labels: true        # Create/update labels
    milestones: true    # Sync as milestones
    comments: false     # Sync progress as comments
  prefix: "[ACT]"       # Prefix for managed issues
  labelPrefix: "act-"   # Prefix for labels
```

## Workflow recommandé

### 1. Initialisation

```bash
# Créer les labels et issues initiales
/act:sync-github --push --labels
```

### 2. Pendant le développement

```bash
# Après chaque phase complétée
/act:sync-github --push
```

### 3. Avant une PR

```bash
# Générer la description de PR
/act:handoff --format github-pr
```

### 4. Collaboration

```bash
# Récupérer les changements faits sur GitHub
/act:sync-github --pull
```

## Gestion des conflits

Si l'état local et GitHub divergent :

1. **Mode interactif** : ACT affiche les différences
2. **Choix** : Local wins (`--push --force`) ou Remote wins (`--pull --force`)
3. **Merge** : ACT propose une fusion intelligente

```bash
# Voir les conflits
/act:sync-github

# Forcer l'état local
/act:sync-github --push --force

# Forcer l'état remote
/act:sync-github --pull --force
```

## Sécurité

- **Token GitHub** : Requis pour push (read/write issues)
- **Permissions** : `repo` scope minimum
- **Rate limiting** : ACT respecte les limites API GitHub
- **Audit** : Toutes les syncs sont loggées dans `progress.md`

### Configuration du token

```bash
# Via variable d'environnement (recommended)
export GITHUB_TOKEN=ghp_xxxxx

# Or use GitHub CLI
gh auth login
```

```yaml
# In .act/config.yaml:
github:
  # token: DO NOT store tokens here — use GITHUB_TOKEN env var or 'gh auth login'
```

---

## Limitations

- **Sync unidirectionnelle préférée** : `--push` est plus fiable que `--pull`
- **Commentaires** : Non synchronisés par défaut (verbeux)
- **Assignees** : Non gérés automatiquement
- **Projects (beta)** : Non supportés dans v2.5
