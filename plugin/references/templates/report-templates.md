## Template usage

### Automatic generation

The `/onboard` skill generates these files in:

```
[PROJECT]/docs/onboarding/
├── AUDIT-[DATE].md      # Complete report
├── CHECKLIST-[DATE].md  # Standalone checklist
└── DIAGNOSTIC.yaml      # Machine-readable data
```

### Variables to replace

| Variable | Description | Example |
|----------|-------------|---------|
| `[PROJECT_NAME]` | Project name | "Bonsai Tracker" |
| `[DATE]` | Short date format | "2026-01-05" |
| `[DATE_ISO]` | ISO 8601 date | "2026-01-05T14:30:00Z" |
| `[X]` | Current phase (1-9) | 6 |
| `[Y]` | Target phase (X+1) | 7 |
| `[XX]` | Numeric score | 75 |

### TodoWrite integration

If the user accepts, checklist items are converted to todos:

```javascript
// Checklist -> TodoWrite conversion
{
  content: "[Task]",
  status: "pending",
  activeForm: "[Task in progress]"
}
```
