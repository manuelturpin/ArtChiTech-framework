# Phase 6: Launch

**Objective**: Deploy and acquire the first users

**Active skills**: `@skills/verification/SKILL.md`

## Checklist

- [ ] Production environment ready
- [ ] Domain and SSL configured
- [ ] Monitoring and alerts active
- [ ] Operations runbook documented
- [ ] Rollback plan tested
- [ ] Landing page optimized
- [ ] Analytics configured
- [ ] Marketing assets ready
- [ ] Launch announcement prepared
- [ ] Support channels configured
- [ ] Onboarding flow tested
- [ ] Go/No-Go documented

## Anti-Patterns

⚠️ **Big bang launch**: Launching everything at once without soft launch.

⚠️ **No rollback plan**: No plan B if things go wrong.

⚠️ **Ignoring metrics**: Launching without knowing what to measure.

⚠️ **Over-promising**: Exaggerated marketing → disappointment.

## Technical Launch Checklist

```
□ DNS propagated
□ SSL valid
□ CDN configured
□ Database backups active
□ Centralized logs
□ PagerDuty/Slack alerts
□ Public status page
□ Rate limiting active
```

## Recommended Tools

- **Vercel/Netlify**: Hosting
- **Cloudflare**: CDN, DDoS protection
- **StatusPage**: Public status
- **Mixpanel/Amplitude**: Product analytics

## Key Metrics

| Metric | Week 1 Target |
|--------|---------------|
| Signups | Defined by market |
| Activation rate | > 30% |
| Uptime | > 99.9% |
| Avg response time | < 200ms |

## Transition → Phase 7

Go Conditions:
- Successful launch
- 0 critical incidents
- First active users
- Initial feedback collected
