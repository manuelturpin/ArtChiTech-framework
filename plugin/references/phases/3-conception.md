# Phase 3 : Conception

**Objectif** : Designer l'architecture technique et UX

**Skills actifs** : `superpowers:writing-plans`, `superpowers:brainstorming`

## Checklist

- [ ] Architecture technique définie
- [ ] Stack technologique choisi
- [ ] Data model conçu
- [ ] API design (REST/GraphQL)
- [ ] User flows documentés
- [ ] Wireframes/Mockups créés
- [ ] Design system initié
- [ ] Security considerations documentées
- [ ] Scalability plan
- [ ] Infrastructure plan (cloud, CI/CD)
- [ ] Technical debt strategy
- [ ] Go/No-Go documenté

## Anti-Patterns

⚠️ **Over-engineering** : Architecturer pour 1M utilisateurs quand vous en aurez 100.

⚠️ **Shiny object syndrome** : Choisir une tech parce qu'elle est nouvelle, pas parce qu'elle convient.

⚠️ **Skipping security** : "On ajoutera la sécurité plus tard".

⚠️ **No design system** : Réinventer les composants à chaque écran.

## Outils Recommandés
