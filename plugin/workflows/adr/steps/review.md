# Step: Review ADR

## Trigger
- ADR created with status "Proposed"
- User requests ADR review

## Process

1. **Completeness check**
   - All sections filled?
   - Context clear enough for someone unfamiliar?
   - Alternatives genuinely considered?
   - Consequences realistic?

2. **Quality check**
   - Decision aligned with project goals?
   - Consequences accurately described?
   - No missing alternatives?

3. **Status update**
   - If approved → Change status to "Accepted"
   - If needs work → Add review notes, keep "Proposed"

## Checklist

- [ ] Context explains the "why" clearly
- [ ] Decision is specific and actionable
- [ ] At least 2 alternatives considered
- [ ] Positive and negative consequences listed
- [ ] References provided where applicable
