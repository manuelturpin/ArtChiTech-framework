# Scoring Rules Reference

## Phase 1: Discovery (0-100)

| Criterion | Points | How to verify |
|-----------|--------|---------------|
| README with problem | 30 | Grep "problem", "why", "objective" |
| Personas/users defined | 25 | Personas file or users section |
| Documented validation | 25 | Interviews, surveys, data |
| Scope defined | 20 | MVP features listed |

## Phase 2: Strategy (0-100)

| Criterion | Points | How to verify |
|-----------|--------|---------------|
| Roadmap | 30 | ROADMAP.md or docs/roadmap |
| Business model | 25 | Pricing, monetization doc |
| Competitors analysis | 25 | Benchmark doc |
| KPIs defined | 20 | Metrics doc |

## Phase 3: Design (0-100)

| Criterion | Points | How to verify |
|-----------|--------|---------------|
| Architecture doc | 30 | docs/architecture or ARCHITECTURE.md |
| Tech stack documented | 25 | Stack in README |
| API specs | 25 | OpenAPI, GraphQL schema |
| UI mockups | 20 | Figma link, wireframes |

## Phase 4: Development (0-100)

| Criterion | Points | How to verify |
|-----------|--------|---------------|
| Source code | 25 | src/ or lib/ not empty |
| Tests present | 25 | tests/ or *.test.* files |
| CI/CD configured | 25 | .github/workflows or .gitlab-ci |
| Code documentation | 25 | JSDoc, docstrings, comments |

## Phase 5: Quality (0-100)

| Criterion | Points | How to verify |
|-----------|--------|---------------|
| Coverage > 60% | 30 | Coverage report |
| No critical bugs | 30 | Issue tracker |
| Clean linting | 20 | ESLint/Prettier pass |
| Performance OK | 20 | Lighthouse, benchmarks |

## Phase 6: Launch (0-100)

| Criterion | Points | How to verify |
|-----------|--------|---------------|
| Deployed | 30 | Production URL |
| Monitoring | 25 | Sentry, logs |
| User documentation | 25 | User guide |
| Launch checklist | 20 | LAUNCH.md complete |

## Phase 7: Growth (0-100)

| Criterion | Points | How to verify |
|-----------|--------|---------------|
| Analytics | 30 | GA, Mixpanel, etc. |
| Feedback collected | 25 | Reviews, NPS |
| Iterations done | 25 | Changelog, releases |
| Growth metrics | 20 | Users, revenue tracked |
